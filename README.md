# ♟ Chess Roguelike

Пошаговый рогалик, где игрок — шахматная фигура, меняющая форму (пешка → конь → слон → ладья → ферзь), спускаясь по процедурным этажам с биомами, врагами, особыми клетками, статусами, лутом, экономикой и мета-прогрессией.

Проект собран под **Vite** (dev-сервер + сборка) с тестами на **Vitest**.

## Запуск

```bash
npm install
npm run dev      # локальный дев-сервер (Vite), обычно http://localhost:5173
npm run build    # прод-сборка в dist/
npm run preview  # предпросмотр собранного билда
npm test         # прогон всех тестов (Vitest)
npm run test:watch
```

## Архитектура

Вся логика вынесена из монолитного HTML в ES-модули `src/`, разделённые по ответственности. Изменяемое состояние забега централизовано в едином объекте `S` (`src/state.js`); модули импортируют его и мутируют — это позволяет свободно резать код на части без «глобалок».

| Модуль | Ответственность |
|---|---|
| `config.js` | Чистые данные-таблицы: CFG (тюнинг), глифы/имена, биомы, тиры редкости, цены, направления-константы. |
| `content.js` | Контент: реликвии, проклятия, достижения, мета-апгрейды. |
| `util.js` | Чистые хелперы: `key/inB/tileColor/cheb/randInt/pick/shuffle/makeForm`, векторы ходов. |
| `state.js` | Объект состояния `S` + предикаты `has/curse` + `enemyAt`. |
| `status.js` | Движок статусов (яд/оглушение/щит/ускорение): `applyStatus/statusVal/cleanse`. |
| `meta.js` | Персистентная мета: осколки, апгрейды, кодекс/бестиарий, достижения, `recordKill`, `endRunMeta`. |
| `board.js` | Генерация комнат по биомам, проходимость, бюджет врагов, спавн, `newFloor/reset`. |
| `moves.js` | Генерация ходов (`genMoves`) для всех фигур + ворота/зоны/ускорение, `effectiveForm`, расчёт угроз. |
| `combat.js` | Ход игрока, взятия/деградация, триггеры клеток, промоушен, мат, конец забега. |
| `enemies.js` | ИИ врагов и спец-поведение (страж/некромант/двойник/ассасин/жрец/маг). |
| `loot.js` | Пул лута, взвешивание по редкости, применение реликвий/проклятий. |
| `events.js` | Комнаты-события: лавка, алтарь очищения, святилище, азарт, благословение. |
| `render.js` | Отрисовка доски/фигур/клеток/статусов на canvas. |
| `ui.js` | Модалки и экраны (титул, лут, сводка, бестиарий, достижения, справка), `syncUI`, лог. |
| `dom.js` | Ссылки на DOM-элементы (`initDom`). |
| `main.js` | Точка входа: ввод (тап/клавиши) и бутстрап. |

Поток забега: `newFloor → ход игрока → ход врагов (afterEnemies) → зачистка → лут → событие → newFloor`. Гибель открывает экран сводки; между забегами копится мета.

## Тесты

`test/*.test.js` (Vitest) покрывают все подсистемы, импортируя модули напрямую и работая с состоянием `S`. `test/setup.js` подставляет заглушки DOM/canvas/localStorage, чтобы движок исполнялся под Node.

- `moves` — генерация ходов, ворота/зоны, ускорение, `effectiveForm`.
- `board` — циклирование биомов, связность всех стилей стен, инварианты баланса.
- `status` — движок статусов и источники.
- `combat` — лестница деградации, броня стража, триггеры клеток, лава.
- `enemies` — поведение шести спец-врагов.
- `loot` — редкость по глубине, уклон проклятых сделок, связки враг↔лут.
- `meta` — начисление/персистентность, кодекс, достижения.
- `events` — золото и все комнаты-события.
- `boot` — приложение стартует без ошибок.

## Дальнейшая разработка

- Добавить врага: тип в `config` (cost/unlock/glyph/name), поведение в `enemies.js`, при желании MOVE_AS/threat в `moves.js`.
- Добавить реликвию/проклятие: запись в `content.js`, тир в `config`, хук-эффект в нужном модуле.
- Добавить клетку: генерация в `board.placeSpecials`, триггеры в `combat`/`enemies`, отрисовка в `render.drawSpecial`.
- Добавить статус: он уже общий — вызывай `applyStatus(unit, k, n)` откуда угодно, добавив цвет в `STATUS_META`.
